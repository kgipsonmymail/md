# Wind 提示词记录

记录 AI 排版提示词的迭代历史和优化过程。

## 版本历史

### v1.0 - 初始版本（2026 年 3 月初）

**文件**：prompt.txt

**内容**：

```markdown
参考 md 公众号渲染语法，为用户给你的文章进行排版润色。

# 探索 Markdown 的奇妙世界

...

```

**特点**：

- 基本语法说明
- 缺少任务目标
- 没有排版框架
- 示例过于简单

**问题**：

- AI 不理解排版目标
- 排版结果不稳定
- 缺少统一风格
- 格式不统一

---

### v2.0 - 优化版本（2026 年 3 月中旬）

**文件**：system_prompt.md

**改进点**：

1. 添加明确的任务目标
2. 提供完整的文章结构框架
3. 添加排版优化规则
4. 提供完整的排版示例

**核心内容**：

#### 任务目标

1. 格式丰富：灵活运用各种 Markdown 格式
2. 结构统一：采用统一的排版风格
3. 条理清晰：通过合理的标题层级、分点、表格等方式
4. 内容完整：保留原文所有信息

#### 文章结构框架

1. 标题（一级标题 #）
2. 日期（粗体格式）
3. 导读（二级标题 ##）
4. 图片（如有）
5. 主要章节（## 一、二、三...）
6. 风险提示（## 风险提示）
7. 作者信息（## 作者信息）
8. 推荐阅读（## 推荐阅读）
9. 免责声明（## 免责声明）

#### 排版优化规则

1. 标题层级优化：添加中文序号
2. 重点内容强调：引用块+粗体
3. 列表优化：长段落改为无序列表
4. 表格使用：整理并列观点
5. 首行缩进：保留全角空格
6. 日期格式：粗体格式
7. 主动分点：理解内容主动添加子标题

#### 优化流程

1. 分析内容结构
2. 添加序号标题
3. 主动分点
4. 强调重点
5. 优化列表
6. 使用表格
7. 优化风险提示
8. 调整结构顺序
9. 检查格式

**效果**：

- 排版质量明显提升
- 结构更加统一
- 格式更加规范
- 风格更加一致

### v2.1 - 细节优化与 Bug 反馈（2026 年 3 月 25 日）

**内容**：
针对网页导入后的颜色样式、图片描述绑定、代码换行等细节进行了反馈与初步修复。

**改进/反馈项**：

1. **已修复**：颜色解析失效问题（橙色变灰色）。通过 `HtmlParser` 的 `getStyleValue` 增强了对 `!important` 和重复属性的处理。
2. **待处理/提示词约束**：
   - 图片名称与图片绑定：目前解析后名称在图片下方，常被误认为带格式正文。
   - 双栏布局兼容性：原文一行两图在 Markdown 中暂无直接双栏对应。
   - 代码框自动换行：目前渲染的代码框不支持自动换行。
   - 异常表情符号：某些高亮块出现多余的 👉 表情，疑为原文本中隐藏字符。
   - 列表样式调节：列表点和数字目前固定为纯黑色。
   - 下划线兼容性：下划线格式有时未能与颜色配置成功叠加。

## 提示词设计原则

### 1. 明确性

- 清晰的任务目标
- 具体的排版要求
- 明确的输出格式

### 2. 完整性

- 包含所有必要的语法说明
- 提供完整的示例
- 覆盖各种使用场景

### 3. 可操作性

- 提供具体的操作步骤
- 给出明确的规则
- 说明注意事项

### 4. 示例驱动

- 提供完整的排版示例
- 展示各种格式的使用
- 说明排版的标准

---

## 提示词结构

### 1. 角色定义

```
你是一位专业的公众号文章排版专家。
```

### 2. 任务目标

```
你的任务是根据用户提供的原始文章，运用 Markdown 语法和排版规则，
将其优化为格式丰富、结构统一、条理清晰的公众号文章。
```

### 3. 文章结构框架

```
标准文章结构（从上到下）：
1. 标题
2. 日期
3. 导读
4. 图片
5. 主要章节
6. 风险提示
7. 作者信息
8. 推荐阅读
9. 免责声明
```

### 4. Markdown 语法说明

- 标题语法
- 字体样式
- 列表语法
- 链接与图片
- 引用语法
- 代码块
- 表格语法
- 分割线
- 风险提示格式
- 免责声明格式

### 5. 排版优化规则

- 标题层级优化
- 重点内容强调
- 列表优化
- 表格使用
- 首行缩进
- 日期格式
- 主动分点原则

### 6. 优化流程

```
1. 分析内容结构
2. 添加序号标题
3. 主动分点
4. 强调重点
5. 优化列表
6. 使用表格
7. 优化风险提示
8. 调整结构顺序
9. 检查格式
```

### 7. 注意事项

- 保留原文内容
- 不改变文章观点
- 保持专业性和可读性
- 确保 Markdown 语法正确
- 保留所有原有链接和图片

### 8. 排版示例

提供完整的排版示例，展示各种格式的使用。

## 提示词优化技巧

### 1. 使用示例

提供完整的排版示例，让 AI 理解期望的输出格式。

### 2. 明确规则

给出明确的排版规则，而不是模糊的描述。

### 3. 分步骤

将复杂的任务分解为多个步骤，逐步完成。

### 4. 强调重点

使用粗体、列表等方式强调重要的规则。

### 5. 提供上下文

说明排版的背景和目标，帮助 AI 理解任务。
