## 回答你的问题

### 1. wind 放根目录还是 apps/web？

**建议：都不单独放。融入现有 monorepo 结构。**

你说得对——之前独立放 wind 目录不好用，因为它和 web 应用的组件、store、构建管道紧密耦合，独立目录导致 import 路径复杂、无法享受 Vite 热更新等。

最佳做法是按职责拆分到已有的包中：

| 内容                       | 放置位置                         | 原因                              |
| -------------------------- | -------------------------------- | --------------------------------- |
| system_prompt、模板配置    | `packages/shared/src/templates/` | 多端共享（web/cli/vscode 都能用） |
| AI 排版增强组件            | `apps/web/src/components/ai/`    | 直接扩展现有 AI 模块              |
| MD 模板备份文件            | `docs/templates/*.md`            | 方便手动复制到其它地方使用        |
| wind/prompt.txt 等开发笔记 | `wind/` 保留                     | 仅作开发备忘，不参与构建          |

### 2. docs 是否统一？

**是的，建议统一到根目录 `docs/`。** 当前 docs/ 已有 `custom-upload.md`、`telegram-usage.md` 等，加上你的期货文档和模板文档，统一管理更清晰：

```
docs/
├── guides/              # 原有使用指南
│   ├── custom-upload.md
│   ├── telegram-usage.md
│   └── mp-card.md
├── templates/           # 模板的 MD 备份（可手动使用）
│   ├── finance.md       # 期货/金融类
│   ├── medicine.md      # 中医类
│   ├── activity.md      # 活动类
│   └── personal.md      # 个人类
└── samples/             # 示例文章
    └── 油价分析-山金期货.md
```

### 3. 关于 system_prompt.txt

这是一份非常完整的期货文章排版方案。当前项目中 `ToolBoxPopover.vue` 的 systemPrompt 只有一句话：

```ts
const systemPrompt = `你是一名专业的多语言文本助手，请根据用户的指令处理下列内容...`
```

而你的 `system_prompt.txt` 有 **300+ 行详细的排版规则和示例**，远比现有的强大。我们需要：

- 将它作为**模板系统**的第一个模板（金融/期货类）集成进去
- 让用户可以在 UI 中选择不同的排版模板
- 每个模板都有对应的 `.md` 备份

### 4. wind 目录现状

wind 目录**还在**，里面有 2 个文件：

- `wind/prompt.txt` — 开发笔记（已修改）
- `wind/system_prompt.txt` — 期货排版系统提示词

AI 排版组件也还在 `apps/web/src/components/ai/tool-box/ToolBoxPopover.vue`，没丢。
