# Wind 项目开发进展

## 项目概述

Wind 是基于 doocs/md 项目开发的中间件，专注于公众号文章的 AI 智能排版。项目目标是提供一套完整的文章排版解决方案，包括 Markdown 语法转译、渲染预览、公众号兼容导出和自动上传等功能。

## 开发历程

#### 1.2 核心功能验证

- **排版器使用验证**：成功使用原项目的排版器功能

- **AI 排版验证**：实现 AI 模型调用进行文章排版

### 第二阶段：AI 提示词优化（2026 年 3 月）

#### 2.1 提示词设计

- **初版提示词**：基于 Markdown 公众号渲染语法，主要包含语法说明
- **问题识别**：
  - 缺乏完整的任务说明
  - 没有明确的内容框架
  - 缺少排版风格指导

#### 2.2 提示词优化

- **结构优化**：添加完整的文章结构框架

  - 标题
  - 日期
  - 导读
  - 主要章节（带序号）
  - 风险提示
  - 作者信息
  - 推荐阅读
  - 免责声明

- **格式优化**：

  - 标题层级优化（添加中文序号）
  - 重点内容强调（引用块+粗体）
  - 列表优化（长段落改为无序列表）
  - 表格使用规范
  - 首行缩进要求
  - 免责声明格式（小号灰色字体）

- **优化流程**：
  1. 分析内容结构
  2. 添加序号标题
  3. 主动分点
  4. 强调重点
  5. 优化列表
  6. 使用表格
  7. 优化风险提示
  8. 调整结构顺序
  9. 检查格式

#### 2.3 提示词成果

- 创建 system_prompt.md 文件，包含完整的排版规则和示例
- 提示词包含：
  - 任务目标说明
  - 文章结构框架
  - Markdown 语法说明
  - 排版优化规则
  - 优化流程
  - 注意事项
  - 完整排版示例

```
│   ├── 项目结构.md
│   ├── 功能清单.md
│   ├── 错题本.md
│   ├── 提示词记录.md
│   ├── README.md
│   └── 代码规范.md
├── system_prompt.md         # AI 排版系统提示词
├── 初始语法.md              # Markdown 语法参考
└── prompt.txt               # 开发记录（待整理）
```

### 核心功能模块

#### 1. AI 排版模块

- **输入**：原始文章文本
- **处理**：调用 AI API，使用优化后的系统提示词
- **输出**：排版后的 Markdown 文本

#### 2. 图床模块

- **支持平台**：GitHub（已验证）
- **配置方式**：环境变量（.env）
- **功能**：图片上传、CDN 加速

#### 3. 渲染模块

- **基于**：doocs/md 渲染引擎
- **特性**：渲染即所得，无损复制到公众号

## 配置说明

### 环境变量配置

```env
# AI 服务配置
AI_API_URL=https://open.bigmodel.cn/api/paas/v4/chat/completions
AI_API_KEY=your_api_key
AI_MODEL=glm-4.7-flash

# 图床配置（GitHub）
GITHUB_TOKEN=your_github_token
GITHUB_OWNER=kgipsonmymail
GITHUB_REPO=image-home
GITHUB_BRANCH=main
GITHUB_PATH=wind-assets
GITHUB_CDN=true
```

### 默认配置策略

- AI 提示词：默认使用 system_prompt.md
- 图床配置：通过 .env 文件配置，不暴露给用户
- 隐藏"显示配置"按钮，使用环境变量控制

## 开发规范

### 代码规范

1. **不添加注释**：除非明确要求
2. **优先编辑**：优先编辑现有文件，而非创建新文件
3. **遵循约定**：遵循项目现有的代码风格和约定
4. **安全第一**：不暴露敏感信息，使用环境变量

### 文档规范

1. **文档集中管理**：所有文档存放在 wind/docs 目录
2. **避免重复**：通过引用避免内容重复
3. **及时更新**：每次开发完成后更新文档
4. **自动备份**：使用 Git 进行版本管理

### 提交规范

1. **不主动提交**：不主动提交代码到 Git
2. **明确要求**：仅在用户明确要求时提交
3. **测试通过**：确保代码测试通过后再提交
