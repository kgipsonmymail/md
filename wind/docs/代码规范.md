# Wind 代码规范

## 编码规范

### 1. 不添加注释

除非明确要求，否则不添加代码注释。

**示例**：

```javascript
const result = await callAI(prompt)
```

**错误示例**：

```javascript
// 调用 AI 排版
const result = await callAI(prompt)
```

### 2. 优先编辑现有文件

优先编辑现有文件，而非创建新文件。

**原则**：

- 检查是否已有类似功能的文件
- 优先在现有文件中添加功能
- 避免创建重复的文件

### 3. 遵循项目约定

遵循项目现有的代码风格和约定。

**检查项**：

- 代码风格（缩进、命名等）
- 文件组织结构
- 导入导出方式
- 错误处理方式

### 4. 安全第一

不暴露敏感信息，使用环境变量。

**示例**：

```javascript
const apiKey = import.meta.env.VITE_AI_API_KEY
```

**错误示例**：

```javascript
const apiKey = 'sk-xxxxxxxxxxxx'
```

## 文档规范

### 1. 文档集中管理

所有文档存放在 `wind/docs` 目录。

**目录结构**：

```
wind/docs/
├── 项目结构.md
├── 功能清单.md
├── 错题本.md
├── 提示词记录.md
├── 开发进展.md
├── README.md
└── 代码规范.md
```

### 2. 避免重复

通过引用避免内容重复。

**示例**：

```markdown
详见 [功能清单.md](./功能清单.md)
```

### 3. 及时更新

每次开发完成后更新文档。

**更新时机**：

- 完成新功能后
- 修复 Bug 后
- 优化代码后
- 改进流程后

### 4. 文档引用关系

建立清晰的文档引用关系，避免内容重复。

**引用关系**：

```
README.md
    ├── 引用 → 项目结构.md
    ├── 引用 → 功能清单.md
    ├── 引用 → 开发进展.md
    └── 引用 → 代码规范.md

开发进展.md
    ├── 引用 → 功能清单.md
    ├── 引用 → 错题本.md
    └── 引用 → 提示词记录.md
```

## 提交规范

### 1. 不主动提交

不主动提交代码到 Git。

**原则**：

- 仅在用户明确要求时提交
- 避免自动提交
- 让用户控制提交时机

### 2. 明确要求

仅在用户明确要求时提交。

**示例**：

```
用户：请帮我提交代码
助手：好的，正在提交代码...
```

### 3. 测试通过

确保代码测试通过后再提交。

**检查项**：

- 代码能正常运行
- 没有明显的 Bug
- 功能符合预期
- 文档已更新

## 命名规范

### 1. 文件命名

使用中文命名文档文件，使用英文命名代码文件。

**示例**：

```
wind/docs/功能清单.md        # 文档文件
wind/system_prompt.md        # 文档文件
src/utils/format.ts          # 代码文件
src/components/AIFormatter.vue  # 代码文件
```

### 2. 变量命名

使用 camelCase 命名变量。

**示例**：

```javascript
const userName = 'John'
const isLoggedIn = true
```

### 3. 函数命名

使用 camelCase 命名函数，动词开头。

**示例**：

```javascript
function callAI() {}
function uploadImage() {}
function formatContent() {}
```

### 4. 常量命名

使用 UPPER_SNAKE_CASE 命名常量。

**示例**：

```javascript
const MAX_RETRY_COUNT = 3
const API_TIMEOUT = 5000
```

## 代码风格

### 1. 缩进

使用 2 个空格缩进。

**示例**：

```javascript
function formatContent(content) {
  const result = process(content)
  return result
}
```

### 2. 引号

优先使用单引号，仅在需要时使用双引号。

**示例**：

```javascript
const message = 'Hello World'
const json = '{"key": "value"}'
```

### 3. 分号

在语句末尾使用分号。

**示例**：

```javascript
const result = await callAI(prompt)
```

### 4. 空格

在操作符前后添加空格。

**示例**：

```javascript
const sum = a + b
if (condition) {
  doSomething()
}
```

## 错误处理

### 1. 捕获所有错误

使用 try-catch 捕获所有可能的错误。

**示例**：

```javascript
async function callAI(prompt) {
  try {
    const result = await fetch(url, options)
    return result
  }
  catch (error) {
    console.error('调用 AI 失败:', error)
    throw error
  }
}
```

### 2. 友好的错误提示

显示友好的错误提示，而不是技术错误信息。

**示例**：

```javascript
try {
  const result = await callAI(prompt)
}
catch (error) {
  showError('AI 排版失败，请稍后重试')
}
```

### 3. 提供解决方案

在错误提示中提供解决方案。

**示例**：

```javascript
showError('上传失败：文件过大，请压缩后重试')
```

## 性能优化

### 1. 避免重复计算

缓存计算结果，避免重复计算。

**示例**：

```javascript
const cache = new Map()

function expensiveCalculation(input) {
  if (cache.has(input)) {
    return cache.get(input)
  }
  const result = doCalculation(input)
  cache.set(input, result)
  return result
}
```

### 2. 使用异步操作

使用 async/await 处理异步操作。

**示例**：

```javascript
async function uploadImages(files) {
  const results = await Promise.all(
    files.map(file => uploadImage(file))
  )
  return results
}
```

### 3. 防抖和节流

对频繁触发的事件使用防抖和节流。

**示例**：

```javascript
import { debounce } from 'lodash-es'

const handleInput = debounce((value) => {
  console.log('输入:', value)
}, 300)
```

## 安全规范

### 1. 不暴露敏感信息

使用环境变量存储敏感信息。

**示例**：

```javascript
const apiKey = import.meta.env.VITE_AI_API_KEY
```

### 2. 输入验证

验证用户输入，防止 XSS 攻击。

**示例**：

```javascript
function sanitizeInput(input) {
  return input.replace(/</g, '&lt;').replace(/>/g, '&gt;')
}
```

### 3. HTTPS

使用 HTTPS 传输敏感数据。

**示例**：

```javascript
const url = 'https://api.example.com/endpoint'
```

## 测试规范

### 1. 单元测试

为核心功能编写单元测试。

**示例**：

```javascript
describe('formatContent', () => {
  it('should format content correctly', () => {
    const input = 'test content'
    const result = formatContent(input)
    expect(result).toBeDefined()
  })
})
```

### 2. 集成测试

为完整流程编写集成测试。

**示例**：

```javascript
describe('AI Formatting Flow', () => {
  it('should format article successfully', async () => {
    const article = 'test article'
    const result = await formatArticle(article)
    expect(result).toHaveProperty('content')
  })
})
```

### 3. 手动测试

在开发过程中进行手动测试。

**测试项**：

- 功能是否正常工作
- 界面是否显示正确
- 错误是否正确处理
- 性能是否满足要求

## Git 规范

### 1. 分支管理

使用功能分支开发。

**分支命名**：

- `feature/功能名称`：新功能
- `fix/问题描述`：Bug 修复
- `docs/文档更新`：文档更新

### 2. 提交信息

使用清晰的提交信息。

**格式**：

```
类型(范围): 描述

类型：
- feat: 新功能
- fix: Bug 修复
- docs: 文档更新
- style: 代码格式调整
- refactor: 代码重构
- test: 测试相关
- chore: 构建/工具相关
```

**示例**：

```
feat(ai): 添加 AI 排版功能
fix(upload): 修复图片上传失败问题
docs(readme): 更新 README 文档
```

### 3. .gitignore

确保敏感信息不被提交。

**示例**：

```
.env
.env.local
node_modules/
dist/
*.log
```

```javascript
const result = await callAI(prompt)
```

### 2. 错误提示

显示友好的错误提示。

**示例**：

```javascript
showError('排版失败，请稍后重试')
```

### 3. 成功提示

显示成功提示，确认操作完成。

**示例**：

```javascript
showSuccess('排版完成！')
```

对危险操作显示确认对话框。

```javascript
if (confirm('确定要覆盖编辑器内容吗？')) {
```
